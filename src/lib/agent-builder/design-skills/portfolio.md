# Portfolio / Showcase Design

You're designing a portfolio — the WORK is the hero, not the chrome.

## Core Principles

- **Let the work speak.** Minimal UI chrome. Maximum space for showcased projects.
- **Generous whitespace.** Space between projects creates room to breathe and appreciate each piece.
- **Subtle personality.** The designer's taste shows through LAYOUT and SPACING choices, not decorative elements.
- **Quality over quantity.** Better to show 4 amazing projects than 12 mediocre ones.

## Structure

- **Intro**: Name + role + one-sentence positioning. Not a biography — a signal.
- **Project grid**: Large images or thumbnails. Asymmetric grid preferred (not uniform cards).
- **Project detail**: Full-width hero image + brief description + role + outcome.
- **Contact**: Simple and direct. Email, select social links. No contact forms.

## Layout Patterns

- **Masonry**: Mixed-size project thumbnails. Some landscape, some portrait. Creates visual interest.
- **Full-bleed gallery**: Each project gets full-width treatment. Scroll through one at a time.
- **Minimal grid**: 2 columns, generous gap-8 or gap-12. Clean and orderly.

## Rules

- NO stock photos. Use `https://placehold.co/WxH` as project placeholders with aspect ratios that suggest real work (16:9, 4:3, 1:1).
- Headings should be understated: text-sm uppercase tracking-widest, or text-xl font-light. Not bold hero headings.
- Navigation: Minimal. Name/logo left, 2-3 links right. Maybe just "Work" and "About."
- Colors: Monochromatic or very restrained. The projects provide the color.
