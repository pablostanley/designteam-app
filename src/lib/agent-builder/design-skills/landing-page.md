# Landing Page Design

You're designing a landing page. The goal is CONVERSION — every element should move the visitor toward the CTA.

## Structure

1. **Hero** (above the fold): Headline + subtext + CTA. This is 80% of the page's impact. Nail it.
2. **Social proof**: Logos, testimonials, or metrics. Place EARLY (right after hero).
3. **Features/Benefits**: Show what the product does, framed as user outcomes (not feature lists).
4. **Objection handling**: FAQ, comparison, or trust signals to remove doubt.
5. **Final CTA**: Repeat the main call-to-action. Different framing than the hero.

## Rules

- ONE primary CTA per page. Secondary actions are fine but visually subordinate.
- Hero headline: benefit-driven, not feature-driven. "Ship 10x faster" not "AI-powered design tool."
- Above the fold = headline + CTA + one visual. Don't bury the lead.
- Social proof is MANDATORY. Even "Trusted by 500+ teams" is better than nothing.
- Every section should answer "why should I care?" If it doesn't, cut it.

## Layout

- Hero: py-32 minimum. Text max-w-3xl or max-w-4xl. CTA buttons prominent.
- Feature sections: Alternate between centered grid and asymmetric text+image.
- Testimonials: Use real names, real companies. Blockquote with attribution.
- Final CTA: Full-width colored section. Different visual weight than rest of page.
